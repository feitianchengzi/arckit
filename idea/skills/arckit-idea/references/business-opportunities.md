# 商机字段规范

## 主索引格式（`arckit/idea/business-opportunities/business-opportunities.md`）

每行一条，格式：
```
- **{商机名称}** - 客户：{客户名称} - 状态：{状态} - 优先级：{优先级} - 最后更新：{日期}
```

排序规则：按状态优先级排序（项目进行中 > 需求确认 > 前期接洽 > 待跟进 > 已完成 > 已暂停），同状态内按优先级排（高 > 中 > 低）。

## 详情文件字段

| 字段 | 说明 |
|------|------|
| 商机名称 | 简洁，通常含客户名称或项目特征 |
| 客户名称 | 客户公司或个人名称 |
| 客户联系人 | 姓名和职位 |
| 联系方式 | 电话/邮箱等 |
| 商机状态 | 前期接洽 / 需求确认 / 项目进行中 / 已完成 / 已暂停 / 待跟进 |
| 优先级 | 高 / 中 / 低（根据客户重要性和项目价值判断） |
| 创建时间 | 首次创建日期 |
| 最后更新 | 最近一次修改日期 |

## 详情文件章节结构

1. **基本信息** — 上述字段
2. **商机描述** — 客户背景、需求概述
3. **需求管理** — 当前需求 + 历史需求变更
4. **沟通记录** — 按日期倒序的客户沟通摘要
5. **下一步行动** — 待办事项列表（checkbox 格式）
6. **备注** — 其他相关信息
7. **更新记录** — 时间戳 + 变更说明（追加，不覆盖）

## 更新操作规则

- 追加到「更新记录」章节（带日期），不覆盖历史
- 状态变更时同步更新主索引对应条目
- 沟通记录按日期倒序插入
