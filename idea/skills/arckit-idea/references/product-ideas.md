# 产品创意字段规范

## 主索引格式（`arckit/idea/product-ideas/product-ideas.md`）

每行一条，按时间倒序（最新在前），格式：
```
- **{创意名称}** - {一句话描述} - 状态：{状态} - {日期}
```

## 详情文件字段

| 字段 | 说明 |
|------|------|
| 创意名称 | 简洁，便于识别，避免与现有创意重名 |
| 状态 | 探索中 / 进行中 / 已暂停 / 已完成 / 已放弃 |
| 优先级 | 高 / 中 / 低（根据描述判断） |
| 创建时间 | 首次创建日期 |
| 最后更新 | 最近一次修改日期 |

## 详情文件章节结构

1. **基本信息** — 上述字段
2. **产品描述** — 核心想法、解决的问题
3. **核心功能** — 功能点列表（从用户输入提取）
4. **目标用户** — 用户群体描述
5. **市场价值** — 商业价值评估
6. **技术可行性** — 高 / 中 / 低（基于功能复杂度评估）
7. **开发计划** — 阶段性任务（checkbox 格式）
8. **备注** — 其他相关信息
9. **更新记录** — 时间戳 + 变更说明（追加，不覆盖）

## 更新操作规则

- 追加到「更新记录」章节（带日期），不覆盖历史
- 状态变更时同步更新主索引对应条目
- 内容聚焦产品分析，不包含技术实现细节
