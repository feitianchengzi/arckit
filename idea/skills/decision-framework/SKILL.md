# Decision Framework - 决策思考工具箱

## 简介

提供三套递进式思考工具，帮助在产品决策、战略规划、方案评审等场景中提升决策质量。

**核心能力：**
1. **第一性原理拆解** - 找方向（从零重构问题）
2. **产品价值评估** - 判价值（定量评估方案）
3. **苏格拉底追问** - 拷逻辑（六层质疑验证）

---

## 触发条件

当用户有以下需求时触发此 skill：

- "帮我评估这个方案"
- "这个需求值不值得做"
- "帮我想想这个问题"
- "拷问一下这个方案"
- "我们是不是问错问题了"
- "帮我从第一性原理分析"
- "用苏格拉底式提问帮我梳理"
- "做一个产品价值评估"

## 工作流程

### 第一步：场景识别

根据用户描述，自动判断应该用哪个工具：

```
问题定义不清晰？
    ↓ 是
第一性原理拆解
    ↓ 否
有多个候选方案？
    ↓ 是
产品价值评估
    ↓ 否
有具体方案需要验证？
    ↓ 是
苏格拉底追问
```

**判断依据（参考 templates/README.md 第五章）：**

| 用户场景 | 优先工具 |
|---------|---------|
| 全新领域/业务、战略转型 | 第一性原理 |
| "竞品都在做，要不要跟？" | 第一性原理 |
| 需求评审、立项决策 | 产品价值评估 |
| 有 2+ 个候选方案要选择 | 产品价值评估 |
| 方案评审、风险识别 | 苏格拉底追问 |
| 方案反复被否、团队陷入僵局 | 苏格拉底追问 → 第一性原理 |

---

### 第二步：工具选择确认

**自动路由：**
- 根据场景判断，告诉用户"这个场景适合用【工具名】"
- 简要说明为什么（1-2 句话）
- 询问是否确认，或用户想换其他工具

**示例回复：**
> 根据你的描述，这是一个"竞品都在做，要不要跟"的场景。我建议先用**第一性原理拆解**，从零重新审视这个问题，避免盲目跟随。
> 
> 要开始吗？或者你想直接用产品价值评估/苏格拉底追问？

**手动指定（用户明确要求）：**
- "用第一性原理帮我分析" → 直接用第一性原理
- "评估一下值不值得做" → 直接用产品价值评估
- "拷问这个方案" → 直接用苏格拉底追问

---

### 第三步：创建工作文件

确认工具后，立即创建填写文件：

**文件命名规则：**
```
{项目名/主题}-{工具简称}-{日期}.md

工具简称：
- first-principles (第一性原理)
- value-assessment (产品价值评估)
- socratic (苏格拉底追问)

示例：
- 直播带货-first-principles-20250521.md
- AI助手功能-value-assessment-20250521.md
- 开发效率方案-socratic-20250521.md
```

**操作：**
1. 询问用户项目/主题名称（如果用户已在问题中提及，直接提取）
2. 复制对应模版到新文件：
   ```bash
   cp skills/decision-framework/templates/{模版名}.md \
      workspace/{项目名}-{工具简称}-{日期}.md
   ```
3. 告知用户文件已创建，开始引导填写

---

### 第四步：逐章节引导填写

**引导原则：**
- 不要一次性抛出整个模版（用户会懵）
- 逐章节引导，每次只问当前章节的问题
- 用对话式语言，不要照搬模版里的表格结构
- 根据用户回答，帮助梳理并填入对应位置
- 必要时提供案例或提示

**引导流程：**

#### 如果用第一性原理拆解：

1. **〇、原始问题**
   - "先说说你最初想解决的问题或想做的事？"
   - 用户回答后，追问："这个问题背后隐含了哪些假设？比如'必须用某种方式'或'某个前提成立'？"
   - 填入 0 章

2. **一、拆解到最底层**
   - "我们往上追问：为什么要做这个？解决了能得到什么？"（连续追问 3-4 层，直到找到核心目标）
   - "有哪些物理/经济/人性规律是绕不过去的？"
   - "我们手上有哪些基本材料/工具可以直接用？"（人力/技术/数据/渠道/资金）
   - 填入第一章 1.1-1.3

3. **二、重新定义问题**
   - 基于刚才的拆解，帮用户重新表述问题
   - 区分硬约束（必须满足）和软约束（行业惯例，可挑战）
   - 填入第二章

4. **三、从零重构方案**
   - "基于刚才的底层真相，我们能组装出哪些可能的路径？"
   - 引导用户想出 3-5 个路径，每个都要说清楚核心思路
   - 粗估每个路径的成本和可行性
   - 填入第三章

5. **四、挑战行业假设**
   - "这个领域有哪些'大家都这么做'的惯例？"
   - 逐个拆解：为什么大家都这么做？这个假设还成立吗？
   - 填入第四章

6. **五、极简验证**
   - 从第三章的路径中，提取最关键的假设
   - "用什么最低成本的方式验证这些假设？"
   - 填入第五章

7. **六至八章快速过**
   - 帮用户整理对比矩阵、评估壁垒、输出建议
   - 填入第六至八章

#### 如果用产品价值评估：

1. **一、基本信息** → 直接填写
2. **二、核心假设句式**
   - 逐个引导填写六个模块（用户/场景/问题/现有方案/核心方法/价值）
   - 最后汇总成完整句式
3. **三、六维拆解**
   - 逐维度引导（3.1 用户 → 3.2 场景 → 3.3 痛点 → ...）
   - 每个维度填完后让用户自评（聚焦度/场景价值/痛点强度等）
4. **四、商业验证**
   - 如果是立项前，说明这部分 MVP 后再填
   - 如果已上线，引导用户回填实际数据
5. **五、综合评估**
   - 帮用户打分，计算加权总分
   - 输出建议（通过/有条件通过/需修正/推翻）

#### 如果用苏格拉底追问：

1. **〇、命题声明** → 让用户写下要拷问的命题
2. **一、澄清概念**
   - "这个命题里的关键词是什么？分别指什么？"
   - "哪些是事实？哪些是判断？"
3. **二、探查假设**（最关键）
   - "这个论点背后默认了什么？"
   - 列出所有隐含假设，逐个问："为什么可以这样默认？"
   - 评估每个假设的脆弱度
4. **三、审视证据**
   - "逻辑链条是怎样的？从 A 到 B 必然成立吗？"
   - "有没有反面证据？"
5. **四、替代观点**
   - "反对者会怎么说？"
   - "有没有中间立场或更复杂的模型？"
6. **五、检验后果**
   - "如果按这个逻辑推演下去，会产生什么后果？"
   - "有没有不可逆的风险？"
7. **六、反诘问题**（最容易被跳过，但最重要）
   - "我们是否应该问一个更根本的问题？"
   - "这个问题本身是否隐含了错误的预设？"
8. **七、综合评估**
   - 汇总各层结论
   - 判断：通过/修正/推翻/需重新定义问题

---

### 第五步：输出与后续建议

**填写完成后：**

1. **总结关键发现**（3-5 条要点）
2. **明确结论**
   - 第一性原理 → 推荐哪个路径，需要验证什么
   - 产品价值评估 → 总分多少，建议通过/修正/推翻
   - 苏格拉底追问 → 命题状态（通过/修正/推翻/重新定义）
3. **建议下一步**
   - 如果用了第一性原理，且产出了多个方案 → 建议接下来用产品价值评估选出最优
   - 如果用了产品价值评估，且方案通过 → 建议用苏格拉底追问再拷问一遍
   - 如果苏格拉底追问发现问题可能问错了 → 建议用第一性原理重新拆解
4. **存档提醒**
   - 告知填好的文件路径
   - 建议定期复盘更新（尤其是产品价值评估的第四章"商业验证"）

## 三个工具详解

### 工具 1：第一性原理拆解

**何时使用：**
- 全新领域/业务
- 战略转型
- 打破僵局（团队陷入困境）
- "竞品都在做，要不要跟？"

**核心方法：**
1. 拆解到不可改变的基本事实（物理/经济/人性规律）
2. 挑战所有"理所当然"的假设
3. 从零重构多个可能的路径

**产出：**
- 重新定义的问题
- 3-5 个候选方案
- 每个方案的成本与可行性估算

**模版位置：** `templates/first-principles-thinking.md`

---

### 工具 2：产品价值评估

**何时使用：**
- 需求评审
- 立项决策
- 有多个候选方案需要选择

**核心方法：**
六维拆解（用户/场景/痛点/现有方案/核心方法/预期价值）+ 商业定量验证（TAM/LTV/CAC）

**产出：**
- 定量评分（满分 5.0）
- 决策建议（通过/有条件通过/需修正/推翻）
- 关键风险清单

**模版位置：** `templates/product-value-assessment.md`

---

### 工具 3：苏格拉底追问

**何时使用：**
- 方案评审
- 风险识别
- 团队认知对齐
- 方案反复被否（需要找漏洞）

**核心方法：**
六层追问：概念澄清 → 假设探查 → 证据审视 → 替代观点 → 后果检验 → 反诘问题本身

**产出：**
- 命题状态判断（通过/修正/推翻/需重新定义问题）
- 待验证假设清单
- 关键风险

**模版位置：** `templates/socratic-questioning.md`

## 组合使用模式

参考 `templates/README.md` 第二、三章。

### 模式 A：完整严谨流程（重大决策）
```
第一性原理(2-3天) → 产品价值评估(1-2天) → 苏格拉底追问(半天)
```

### 模式 B：快速验证流程（常规需求）
```
产品价值评估 → 苏格拉底追问
```

### 模式 C：深度质疑流程（打破僵局）
```
苏格拉底追问 → 第一性原理
```

---

## 注意事项

1. **不要教条** - 三个工具是思维框架，不是流程规范，按需选择
2. **逐章节引导** - 不要一次性抛出整个模版，用对话式逐步填写
3. **重点关注关键章节**
   - 第一性原理：第一章（拆解到底层）、第四章（挑战假设）
   - 产品价值评估：第二章（核心假设句式）、第三章（六维拆解）
   - 苏格拉底追问：第二章（探查假设）、第六章（反诘问题）
4. **提供案例** - 必要时参考 README.md 第四章的实战案例
5. **建议后续工具** - 完成一个工具后，判断是否需要组合使用
6. **存档与迭代** - 提醒用户保存填好的文件，定期复盘更新

## 示例对话流程

**用户：** "帮我评估一下，要不要做直播带货？"

**Agent：**
> 这是一个"竞品都在做，要不要跟"的场景。我建议先用**第一性原理拆解**，从零审视这个问题，避免盲目跟随。
> 
> 要开始吗？或者你想直接用产品价值评估？

**用户：** "好，开始吧"

**Agent：**
> 我会帮你创建一个工作文件，然后逐步引导你填写。
> 
> 先给这次分析起个名字吧，比如"直播带货可行性分析"？

**用户：** "就叫'直播带货'"

**Agent：**
> 好，文件已创建：`workspace/直播带货-first-principles-20250521.md`
> 
> 现在开始第一步：先说说你最初想解决的问题或想做的事？

**用户：** "我们想提高销售额，看到竞品都在做直播带货，所以考虑要不要跟"

**Agent：**
> 明白了。这个问题背后隐含了几个假设：
> 1. 提高销售额 = 增加新的销售渠道
> 2. 直播带货是有效的销售渠道
> 3. 竞品在做 = 我们也应该做
> 
> 这些假设都成立吗？我们接下来会逐个拆解。
> 
> 现在往上追问：为什么要提高销售额？解决了能得到什么？

**用户：** "公司需要增长，投资人有压力"

**Agent：**
> 继续往上：增长的本质是什么？是营收数字，还是利润，还是市场份额？

**用户：** "主要是利润"

**Agent：**
> 好，找到核心目标了：**提高利润**。
> 
> 那么问题就变成了：直播带货真的能提高利润吗？还是只是提高营收但利润率更低？
> 
> 接下来我们拆解基本事实：有哪些物理/经济/人性规律是绕不过去的？

（继续逐步引导填写...）
