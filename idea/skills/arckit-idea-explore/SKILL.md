# ArcKit Idea Explore — 深度创意探索与可视化

管理 `arckit/idea-explore/` 下的创意探索文档与可视化产物。核心职责：引导 **模糊想法 → 结构化假设 → 验证决策 → 可视化线框图**。

## 与 arckit-idea 的区别

| 维度 | arckit-idea | arckit-idea-explore |
|------|-------------|---------------------|
| 产物 | 单个 `idea.md` | `idea.md` + `idea.html` 线框图 |
| 存储 | `arckit/idea/{idea-name}.md` | `arckit/idea-explore/{idea-name}/` 独立文件夹 |
| 触发 | Agent 自动识别 | 手动 `/arckit-idea-explore` |
| 定位 | 轻量日常记录 | 深度探索 + 可视化输出 |

## 目录结构

```
arckit/idea-explore/
├── {idea-name}/              # 每个创意独立文件夹（kebab-case）
│   ├── idea.md               # 结构化创意文档
│   └── idea.html             # 可视化线框图
└── {another-idea}/
    ├── idea.md
    └── idea.html
```

## 工作流

### 阶段 0：初始化

```
1. 从用户输入提取创意描述和可选的自定义 idea 名称（如 "探索健身追踪应用" → idea-name: fitness-tracker）
2. 确定 IDEA_DIR: arckit/idea-explore/{idea-name}/
3. 确定 IDEA_FILE: arckit/idea-explore/{idea-name}/idea.md
4. 确定 IDEA_HTML: arckit/idea-explore/{idea-name}/idea.html
5. 判断工作模式:
   - IDEA_FILE 不存在 → 创建模式（从头探索）
   - IDEA_FILE 存在   → 更新模式（继续迭代）
```

### 阶段 1：创意探索对话（角色：敏捷产品教练）

**对话原则：**
- 提问优先：通过提问引导思考，而非直接给出答案
- 聚焦核心：用户偏离时主动拉回 — "让我们先聚焦核心假设 X"
- 成本意识：所有建议标注预估时间/成本

**步骤 1a — 假设提炼：**
```
1. 从用户描述识别核心想法
2. 主动提问澄清目标用户、核心痛点
3. 提炼 2-3 个关键假设（按优先级排序）
4. 引导聚焦: "哪个假设最核心？如果不成立，整个创意就不成立？"
```

**步骤 1b — 竞品分析（模拟前进行）：**
```
1. 快速识别 2-3 类现有解决方案
2. 分析各方案优劣势
3. 提炼差异化机会点（此分析用于反方模拟）
4. 引导: "现有方案的优势是什么？我们的差异化点在哪里？"
```

**步骤 1c — AI 模拟用户研究：**
```
正方用户（3-5 人）：支持理由、需求痛点、使用场景
反方用户（2-3 人）：质疑理由、顾虑担忧、替代偏好（参考竞品优势）
中立观察者（1-2 人）：客观分析机会和风险

支持率 = 正方人数 / (正方 + 反方) × 100%

后续建议：
  支持率 ≥ 60% → 「继续探索」
  支持率 40-60% → 「调整方向」
  支持率 < 40%  → 「放弃」
```

**对话检查点（每步骤后确认）：**
- [ ] 假设清晰且可验证（而非模糊猜测）
- [ ] 竞品分析完成（2-3 类现有方案）
- [ ] 模拟完整（正方 3-5 人 / 反方 2-3 人 / 中立 1-2 人）
- [ ] 支持率计算正确
- [ ] 后续建议明确（继续 / 调整 / 放弃）

### 阶段 2：生成结构化文档

读取模板 [assets/idea-template.md](assets/idea-template.md)，根据对话和模拟结果填充。

**必填章节（所有情况）：**
- 一、创意概述
- 二、核心假设（2-3 个，按优先级）
- 三、竞品分析
- 四、用户研究模拟（含支持率和后续建议）

**条件章节（根据后续建议）：**

| 建议 | 文档状态标注 | 五、六、七章 | 说明 |
|------|-------------|------------|------|
| 继续探索 | 🟢 探索中 | 生成完整 MVP / 行动计划 / 风险 | |
| 调整方向 | 🟡 需要调整 | 生成，MVP 反映调整方向，行动首要任务为"调整" | |
| 放弃 | 🔴 不建议继续 | 不生成，四章后添加存档说明 | |

所有情况：**八、探索记录**（含日期）

**更新模式规则：**
- 追加到「八、探索记录」（带日期）
- 模拟结论改变 → 更新第四章
- 新增假设 → 更新第二章
- 发现新竞品 → 更新第三章
- 状态改变（如从放弃改为调整）→ 补充缺失章节

写入 IDEA_FILE。

### 阶段 3：生成可视化线框图

读取模板 [assets/idea-html-template.html](assets/idea-html-template.html)，填充以下 5 个部分：

```
1. 用户旅程（4 步）: 发现问题 → 接触产品 → 开始使用 → 获得价值
2. 价值主张画布: 用户痛点（3-5）/ 产品价值 / 用户需求 / 产品功能
3. 用户研究模拟: 正方卡片 / 反方卡片 / 中立卡片 / 支持率统计 / 模拟结论
4. MVP 功能卡片: 按 P1/P2/P3 优先级列出（如适用）
5. 核心界面草图: HTML + CSS 实现，4-8 个界面（落地页 + P1 功能界面）
   - 严禁使用 ASCII 文本图或 Markdown 代码块绘图
   - 使用 .wireframe-canvas / .device-frame / .nav-bar / .content-area / .list-item 等样式类
```

**占位符替换：**
- `{{IDEA_NAME}}` → 创意名称（从 idea.md 提取）
- `{{DATE}}` → 当前日期（YYYY-MM-DD）
- `{{IDEA_DIR}}` → 文件夹路径（arckit/idea-explore/{idea-name}）

**状态横幅（根据建议）：**
- 继续探索 → `✅ 建议继续 - 创意价值得到验证（支持率 N%）`
- 调整方向 → `⚠️ 需要调整 - 请关注反方用户反馈（支持率 N%）`
- 放弃 → `❌ 不建议继续 - 此创意未通过验证（支持率 N%）`

写入 IDEA_HTML。

### 阶段 4：质量验证

**文档完整性：**
- [ ] 必填四章完整
- [ ] 条件章节与建议一致（放弃时无五六七章）
- [ ] 支持率与建议匹配（继续 ≥ 60%，放弃 < 40%）
- [ ] 更新模式：探索记录已追加

**HTML 可视化：**
- [ ] 状态横幅正确（含支持率）
- [ ] 5 个部分完整
- [ ] 界面草图使用 HTML 实现（4-8 个）
- [ ] 无占位符残留

### 阶段 5：输出报告

```
✅ 创意探索完成！

🎯 AI 模拟建议：【继续探索 / 调整方向 / 放弃】
   理由：[简短说明]
   支持率：N%（正方 N 人 / 反方 N 人）

📂 产物位置：
   arckit/idea-explore/{idea-name}/
   ├── idea.md    ← 结构化创意文档
   └── idea.html  ← 可视化线框图（浏览器打开查看）
```

## 状态标识

```
🟢 继续探索（支持率 ≥ 60%）
🟡 需要调整（支持率 40-60%）
🔴 不建议继续（支持率 < 40%）
⚪ 探索中（尚未完成模拟）
```
