# 思考工具箱使用指南

> 三套模版的定位、关系与使用场景全景图

---

## 一、核心定位

| 模版 | 解决什么问题 | 核心方法 | 典型产出 |
|------|------------|---------|---------|
| **第一性原理拆解** | 我们在解决正确的问题吗？ | 拆到最底层,从零重构 | 全新视角/颠覆性方案 |
| **产品价值评估** | 这个方案值得做吗？ | 六维拆解 + 商业验证 | 立项决策依据 |
| **苏格拉底追问** | 这个方案想清楚了吗？ | 六层质疑,找假设漏洞 | 风险清单/待验证项 |

## 二、递进关系

### 2.1 线性递进模式（完整流程）

```
┌─────────────────────────────────────────────────────────────┐
│                      遇到新问题/新机会                          │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 1：找方向（第一性原理拆解）                                 │
├─────────────────────────────────────────────────────────────┤
│  目标：确保我们在解决正确的问题                                   │
│  ────────────────────────────────────────────────────────   │
│  • 拆解到不可改变的基本事实                                      │
│  • 挑战所有"理所当然"的假设                                     │
│  • 从零重构多个可能的路径                                        │
│  • 不参考任何现有方案                                           │
│  ────────────────────────────────────────────────────────   │
│  产出：3-5个候选方案 + 重新定义的问题                             │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 2：判价值（产品价值评估）                                   │
├─────────────────────────────────────────────────────────────┤
│  目标：从候选方案中选出最值得做的                                  │
│  ────────────────────────────────────────────────────────   │
│  • 目标用户是否足够聚焦？                                        │
│  • 场景是否高频刚需？                                           │
│  • 痛点强度够不够？                                             │
│  • 现有方案的致命不足是什么？                                    │
│  • 我们的方案有什么独特壁垒？                                    │
│  • 价值能量化吗？用户愿意付费吗？                                 │
│  ────────────────────────────────────────────────────────   │
│  产出：选定方案 + 评分矩阵 + 关键风险                             │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  阶段 3：拷问逻辑（苏格拉底追问）                                 │
├─────────────────────────────────────────────────────────────┤
│  目标：确保选定的方案逻辑严密、假设可验证                           │
│  ────────────────────────────────────────────────────────   │
│  • 核心概念定义清楚了吗？                                        │
│  • 有哪些默认假设？它们成立吗？                                  │
│  • 逻辑链条从A到B必然吗？                                       │
│  • 反对者会怎么论证？                                           │
│  • 推演到极限,后果可接受吗？                                     │
│  • 我们是否在解决一个错误的问题？                                 │
│  ────────────────────────────────────────────────────────   │
│  产出：通过/修正/推翻 + 待验证假设清单                            │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
          ┌───────┴───────┐
          │               │
      通过              不通过
          │               │
          ▼               ▼
        执行        回到阶段1重新拆解
```

---

### 2.2 关键决策点

```
问题定义阶段
    │
    ├─→ 清晰明确              → 跳过第一性原理,直接评估
    └─→ 模糊/可能问错了        → 必须先用第一性原理拆解

方案选择阶段
    │
    ├─→ 只有一个方案          → 直接苏格拉底追问
    ├─→ 有多个候选            → 先用产品价值评估筛选
    └─→ 完全没头绪            → 回到第一性原理

逻辑验证阶段
    │
    ├─→ 团队认知一致          → 苏格拉底追问走形式即可
    ├─→ 有分歧/不确定性强      → 必须完整走六层追问
    └─→ 追问后发现假设不成立   → 回到第一性原理/价值评估
```

## 三、三种组合模式

### 模式A：完整严谨流程（重大决策）

**适用场景：** 战略转型、新业务立项、重大投资决策

```
第一性原理（2-3天） → 产品价值评估（1-2天） → 苏格拉底追问（半天）
```

**典型时间轴：**
- Day 1-2: 第一性原理拆解工作坊,产出3-5个候选方案
- Day 3-4: 每个方案分别填写产品价值评估模版
- Day 5: 决策会议,用苏格拉底追问拷问排名前2的方案
- Day 6: 最终决策 + 行动计划

---

### 模式B：快速验证流程（常规需求）

**适用场景：** 功能迭代、优化需求、已有明确方向的项目

```
产品价值评估 → 苏格拉底追问
```

**跳过第一性原理的前提：**
- ✅ 问题定义清晰
- ✅ 大方向已确定
- ✅ 只是在验证具体方案

**典型时间：** 半天到1天

---

### 模式C：深度质疑流程（打破僵局）

**适用场景：** 团队陷入僵局、方案反复被否、长期没突破

```
苏格拉底追问（发现问题可能问错了） → 第一性原理（重新定义问题）
```

**信号：**
- 方案改了N版还是不满意
- 团队对问题理解差异很大
- 竞品都在做但效果都不好

**典型过程：**
1. 先用苏格拉底追问拷问当前方案
2. 如果发现第六层"反诘问题本身"暴露了根本问题
3. 立即切换到第一性原理,重新拆解
4. 拆解后可能发现要做的是完全不同的事

## 四、实战案例对比

### 案例1：是否做直播带货？

#### 路径A：直接评估（常见错误）

```
产品价值评估
    ↓
得分偏低（用户聚焦度3分,场景频次3分）
    ↓
但因为"竞品都在做"还是硬上
    ↓
投入大量资源,效果不佳
```

#### 路径B：完整流程（正确做法）

```
第一性原理拆解
    ↓
核心目标：让潜在客户看到产品并下单
    ↓
拆解发现：直播只是N种触达方式之一
    ↓
重构出5个路径：直播/短视频/私域/老客推荐/内容种草
    ↓
产品价值评估
    ↓
发现"老客推荐+私域运营"得分最高（成本低/转化高/壁垒强）
    ↓
苏格拉底追问
    ↓
拷问假设："我们的老客真的愿意推荐吗？"
    ↓
发现需要先验证老客满意度和激励机制
    ↓
最终决策：先做老客运营+转介绍机制,直播暂缓
```

### 案例2：如何提高开发效率？

#### 路径A：跳过拆解（常见）

```
产品价值评估
    ↓
"引入AI编程助手"得分4.2,通过
    ↓
苏格拉底追问
    ↓
发现假设："开发慢是因为写代码慢"
    ↓
实际数据：写代码只占20%时间,80%在需求澄清和联调
    ↓
方案推翻
```

#### 路径B：完整流程（正确）

```
第一性原理拆解
    ↓
拆解"效率"：交付价值的速度 = 有效产出 / 总时间
    ↓
拆解时间分布：需求澄清30% + 编码20% + 联调30% + 返工20%
    ↓
重新定义问题："如何减少需求变更和返工？"
    ↓
产品价值评估
    ↓
方案A(AI助手)得分3.5 vs 方案B(需求评审流程优化)得分4.8
    ↓
苏格拉底追问方案B
    ↓
通过,且发现可以先小范围试点验证
    ↓
执行：优化需求评审 + 引入PRD模版
```

## 五、何时用哪个？

### 5.1 决策树

```
开始
  │
  ├─ 问题定义清晰? 
  │   ├─ 是 ──→ 有多个方案?
  │   │           ├─ 是 ──→ 【产品价值评估】选出最优
  │   │           └─ 否 ──→ 【苏格拉底追问】验证逻辑
  │   │
  │   └─ 否 ──→ 【第一性原理】重新定义问题
  │
  └─ 当前方案不满意?
      └─ 是 ──→ 【苏格拉底追问】找漏洞
                    ↓
              发现问题本身可能错了?
                    ↓
              【第一性原理】拆解重构
```

---

### 5.2 快速判断表

| 场景特征 | 优先工具 | 次要工具 |
|---------|---------|---------|
| 全新领域/业务 | 第一性原理 | → 产品价值评估 → 苏格拉底追问 |
| 需求评审 | 产品价值评估 | 如需深挖再用苏格拉底追问 |
| 方案评审 | 苏格拉底追问 | 如发现方向错了再用第一性原理 |
| 团队陷入僵局 | 第一性原理 | — |
| 竞品都在做,要不要跟? | 第一性原理 | → 产品价值评估 |
| 有3个候选方案,选哪个? | 产品价值评估 | → 苏格拉底追问排名前2的 |
| 方案反复被老板否 | 苏格拉底追问 | → 第一性原理(可能问题问错了) |
| 项目复盘 | 苏格拉底追问 | 如需重构方向再用第一性原理 |

## 六、三个工具的本质差异

### 6.1 思维方向

```
第一性原理：  ↓ 向下拆解到底层,再 ↑ 向上重构
产品价值评估： → 横向对比评估(用户/场景/痛点/方案/价值)
苏格拉底追问： ↻ 层层深入质疑(概念→假设→证据→视角→后果→问题本身)
```

### 6.2 认知模式

| 工具 | 认知状态 | 典型心态 |
|------|---------|---------|
| 第一性原理 | 空杯心态 | "忘掉一切既有方案,重新想" |
| 产品价值评估 | 理性评估 | "这个方案到底值多少分？" |
| 苏格拉底追问 | 批判质疑 | "你凭什么这么说？" |

### 6.3 产出性质

| 工具 | 产出 | 确定性 | 迭代性 |
|------|------|--------|-------|
| 第一性原理 | 新方向/新方案 | 低(探索性) | 高(随认知深化需重构) |
| 产品价值评估 | 定量评分 + 决策建议 | 中(依赖假设验证) | 中(验证后回填数据) |
| 苏格拉底追问 | 通过/修正/推翻 | 高(逻辑验证) | 低(一次性质检) |

## 七、实施建议

### 7.1 团队协作

**第一性原理：** 
- 最佳人数：3-7人(太少想法单一,太多难聚焦)
- 建议角色：产品+技术+业务+外部视角(不熟悉行业的人)
- 时长：2-4小时工作坊,分段进行

**产品价值评估：**
- 初稿：产品经理独立完成
- 评审：跨职能团队(产品+技术+运营+商业)
- 重点：第四章"商业定量验证"需要数据支持

**苏格拉底追问：**
- 方式一：由外部人(顾问/其他团队)担任质询者
- 方式二：团队内部轮流扮演"魔鬼代言人"
- 关键：第二层"探查假设"最容易暴露盲区

---

### 7.2 常见陷阱

| 工具 | 常见错误 | 如何避免 |
|------|---------|---------|
| 第一性原理 | 拆解不够彻底,停在"行业惯例"层 | 持续追问"为什么",直到物理/经济/人性规律 |
| 产品价值评估 | 只填假设部分,不回填验证数据 | 原型验证、上线前验证或上线后必须回填第四章"商业验证" |
| 苏格拉底追问 | 只走第1-3层,不挑战问题本身 | 第6层"反诘问题"最重要,别跳过 |

---

### 7.3 时间分配建议

**重大决策(如新业务立项)：**
```
第一性原理：40%  (如总共5天,用2天)
产品价值评估：40% (用2天,每个候选方案半天)
苏格拉底追问：20% (用1天,深度拷问排名前2的方案)
```

**常规需求：**
```
产品价值评估：60% (半天)
苏格拉底追问：40% (2-3小时)
```

**打破僵局：**
```
苏格拉底追问：30% (半天,找问题)
第一性原理：70% (1-2天,重新拆解)
```

1. **不要教条** — 三个工具是思维框架,不是流程规范
2. **按需组合** — 根据问题复杂度灵活选择
3. **持续迭代** — 每次使用后复盘,优化模版
4. **记录案例** — 沉淀典型案例到本README中
