# 行业推理规则库

> 本文档为各行业提供设计决策的推理框架。每个行业的风格、配色、字体、动效、反模式均有据可依，而非凭感觉选择。

---

## 一、SaaS / 企业服务

### 风格倾向

1. **功能优先型（推荐）** — 强调信息层级和操作效率，视觉服务于任务完成。理由：SaaS用户的核心诉求是"用工具解决问题"，视觉应减少认知负荷而非增加视觉刺激。
2. **专业信赖型** — 沉稳克制，强调安全和稳定感。理由：企业采购决策者需要感知到产品的可靠性和专业度，过度设计反而引发不信任。
3. **现代极简型** — 大量留白、精简元素、微交互。理由：B2B用户越来越期待B2C级别的体验，但需要在简洁和功能密度之间找到平衡。

### 配色情绪

- **主色调方向**：深蓝 `#1E3A5F` / 钢蓝 `#2563EB` / 深靛 `#312E81` — 传达专业、可信、稳定
- **Accent策略**：
  - 主accent：亮蓝 `#3B82F6` 或翠绿 `#10B981`（成功/确认操作）
  - 次accent：琥珀 `#F59E0B`（警告）、玫红 `#EF4444`（错误/危险操作）
  - accent用于可交互元素（按钮、链接、选中态），避免大面积使用
- **中性色系**：灰阶层次分明，至少6级灰度确保信息层级清晰
- **避忌**：不要使用超过2个饱和accent色；避免高饱和暖色作为主色

### 字体情绪

- **Display性格**：无衬线、几何或人文主义均可，需要传达"现代且可靠"。推荐：Inter / Plus Jakarta Sans / General Sans
- **Body可读性要求**：
  - 正文字号下限14px（中文12px仅限辅助信息）
  - 行高1.5-1.7
  - 字重对比：标题700 vs 正文400，避免500（看起来模糊）
  - 数据/数字展示建议使用tabular figures（等宽数字）

### 关键效果

- **微交互反馈**：按钮点击缩放 `scale(0.97)` + 状态变化色过渡 `transition: 150ms`
- **骨架屏加载**：避免白屏，使用shimmer效果展示加载状态
- **状态切换过渡**：列表/卡片展开收起的height过渡动画 `300ms ease`
- **Toast通知**：从底部或右上角滑入，3秒自动消失，成功/错误/警告三种样式
- **数据可视化交互**：hover时tooltip + 高亮关联数据点

### 反模式

1. **过度装饰的Landing Page与简陋的Dashboard形成巨大落差** — 前者承诺"精致体验"，后者暴露"工具本质"，用户会感到被欺骗
2. **使用大量动效和3D元素** — 企业用户高频操作场景下，动效会变成干扰，影响操作效率
3. **浅色文字在浅色背景上** — SaaS界面信息密度高，对比度不足直接导致可读性问题，WCAG AA是底线
4. **无边框扁平化到极致** — 失去视觉锚点，用户无法区分可交互与不可交互元素
5. **竞品色完全照搬** — 如果你的蓝和Salesforce的蓝一样，企业客户会觉得你是"廉价替代品"而非"独特选择"

### 参考案例

- **Linear** — 深色主题SaaS的标杆，信息密度与美感的平衡；signature：过渡动画系统
- **Notion** — 极简界面+强大功能；signature：块编辑器视觉语言
- **Vercel Dashboard** — 暗色主题+极简信息架构；signature：Geist字体+闪电logo动效

## 二、E-commerce / 电商

### 风格倾向

1. **转化导向型（推荐）** — 视觉设计服务于购买决策，每个元素都在推动用户向支付页前进。理由：电商的核心指标是转化率，视觉应减少购买路径上的摩擦。
2. **品牌沉浸型** — 品牌故事驱动的视觉体验，适用于高端/设计师品牌。理由：高客单价品类需要情感连接和品牌认同，纯功能导向无法支撑溢价。
3. **社交电商型** — 内容+购物混合，强调UGC和社区感。理由：年轻用户决策链路缩短，"看到即买到"需要内容与商品无缝融合。

### 配色情绪

- **主色调方向**：根据品类决定——
  - 通用：深色 `#1A1A2E` 或品牌色
  - 时尚：黑白为主 + 季节性accent
  - 母婴：柔和粉 `#F8BBD0` / 暖黄 `#FFF3E0`
  - 3C数码：深灰 `#212121` + 科技蓝 `#2196F3`
- **Accent策略**：
  - 核心accent：购买/加入购物车按钮必须高对比、高可见度，推荐橙红 `#FF5722` / 翠绿 `#4CAF50`
  - 促销色：红色 `#F44336` 或橙色 `#FF9800`，仅用于促销/折扣标签
  - 信任色：绿色用于"库存充足""已认证"等信任信号
- **避忌**：不要在商品卡片上使用过多颜色标签，促销标签最多2种色系

### 字体情绪

- **Display性格**：无衬线为主，需要在不同语言下保持一致性。推荐：Poppins / Montserrat / Noto Sans SC（中文）
- **Body可读性要求**：
  - 商品名称：16px+，字重600+
  - 价格：20px+，字重700，使用tabular figures
  - 描述：14px，行高1.6
  - 关键规则：价格必须一眼识别，字重和尺寸都应高于商品名称

### 关键效果

- **商品图片交互**：hover放大 / 多角度切换 / 色卡切换商品图
- **加入购物车反馈**：按钮状态变化 + 购物车图标弹跳 + 数量badge更新
- **库存/促销紧迫感**：倒计时动画、"仅剩X件"实时更新
- **图片懒加载**：低质量占位图 + 渐进式加载，避免布局抖动
- **筛选器交互**：实时更新结果数，不跳转页面

### 反模式

1. **商品图片被大量文字标签覆盖** — 用户要看的是商品，不是你的促销信息墙
2. **购买路径超过4步** — 每多一步转化率下降15-20%，视觉应引导而非阻断
3. **移动端商品图过小** — 电商移动端占比60%+，商品图至少占屏幕宽度60%
4. **促销价和原价视觉权重相同** — 促销价必须明显大于原价，原价用删除线+灰色
5. **无限滚动无锚点** — 用户浏览50件商品后无法回到第10件，需要"最近浏览"或分页

### 参考案例

- **Apple Store** — 品牌沉浸型标杆；signature：大图+极简文案+无干扰购买路径
- **Nike.com** — 运动品牌电商标杆；signature：全屏视频hero+个性化推荐
- **小红书商城** — 社交电商标杆；signature：UGC内容与商品卡无缝融合

## 三、社交 / 社区

### 风格倾向

1. **内容优先型（推荐）** — 界面退后，让用户内容成为视觉主体。理由：社交产品的价值在UGC，界面设计应是"透明容器"而非"自我表达"。
2. **亲密温暖型** — 圆角、柔和色彩、对话感设计。理由：社交的核心是人与人的连接，冷硬的界面会降低用户的表达欲和互动意愿。
3. **亚文化表达型** — 大胆用色、自定义程度高、视觉有攻击性。理由：垂直社区需要通过视觉语言筛选同好，"看起来像自己人"是第一道门槛。

### 配色情绪

- **主色调方向**：
  - 通用社交：品牌色为主，避免过深的背景色（UGC内容需要突出）
  - 垂直社区：从社区文化中提取颜色（如ACG社区可用二次元配色）
  - IM类：中性色为主（白/浅灰），accent仅用于通知/状态
- **Accent策略**：
  - 交互accent：品牌色或活力色 `#5B5FC7` / `#7C3AED`
  - 通知/红点：标准红 `#EF4444` — 这是用户训练过的行为模式，不要创新
  - 在线状态：绿色 `#22C55E` — 同上，不要用蓝色或其他颜色
  - 表达/创作：温暖色 `#F59E0B` — 激发创作欲
- **避忌**：不要让UI颜色与用户上传内容抢视觉焦点

### 字体情绪

- **Display性格**：圆润、友好、有温度感。推荐：Nunito / Quicksand / 思源黑体 Rounded
- **Body可读性要求**：
  - 信息流中的文字：15-16px，行高1.6+
  - 评论/对话：14px+，需支持长文本阅读
  - 用户名：字重600，与正文区分
  - 时间戳/辅助信息：12px，低对比度灰色
  - 关键规则：中文社交场景下，字间距适当加宽 `0.02em-0.05em`

### 关键效果

- **内容加载动画**：骨架屏或卡片渐入，避免白屏
- **点赞/互动微交互**：心跳动画、粒子扩散、数字递增
- **下拉刷新**：品牌化刷新动画（如Twitter的鸟、微博的彩虹）
- **消息发送动画**：气泡弹出 + 发送按钮状态变化
- **图片/视频预览**：双指缩放、滑动切换，交互要流畅自然

### 反模式

1. **信息流卡片样式过多** — 3种以上卡片样式会让信息流看起来混乱，用户无法建立阅读节奏
2. **强迫用户实名/露脸** — 社交产品中匿名性是很多人的核心需求，设计应尊重隐私层级
3. **互动按钮过小或位置不固定** — 点赞/评论/分享是最高频操作，触控区域至少44x44px
4. **纯白背景+无分隔** — 信息流需要视觉节奏，完全无分隔的白色流会让人视觉疲劳
5. **忽视暗色模式** — 社交产品夜间使用率极高，暗色模式不是可选项

### 参考案例

- **Discord** — 亚文化社区标杆；signature：服务器图标系统+角色颜色
- **即刻** — 中文社区标杆；signature：黄色品牌色+圆形头像+轻松语气
- **Instagram** — 内容优先型标杆；signature：极简UI+内容全屏沉浸

## 四、开发者工具

### 风格倾向

1. **终端原生型（推荐）** — 视觉语言贴近开发者日常环境（终端/IDE），功能信息密度高。理由：开发者用户对"看起来像工具"有天然信任，过度设计反而可疑。
2. **文档驱动型** — 信息架构以文档为核心，代码即UI。理由：开发者习惯从文档获取信息，好的文档体验比好的视觉体验更重要。
3. **极客玩趣型** — 适度使用复古终端/黑客美学，但不过度。理由：开发者社区有独特的文化认同，适当的"内行梗"可以建立归属感。

### 配色情绪

- **主色调方向**：
  - 暗色主题：深蓝灰 `#0F172A` / 深灰 `#1E1E2E` — 不用纯黑
  - 亮色主题：冷白 `#FAFAFA` + 细边框
  - 两套主题都必须是first-class citizen
- **Accent策略**：
  - 代码相关：遵循语法高亮配色体系（如One Dark / Catppuccin / Dracula）
  - 交互accent：蓝 `#3B82F6` / 紫蓝 `#8B5CF6`
  - 状态色：绿=成功/运行 `#22C55E`，红=错误/失败 `#EF4444`，黄=警告 `#EAB308`
  - 避免酸绿和neon色——这是"假极客"的标志
- **避忌**：不要使用渐变按钮、不要使用3D卡片效果、不要使用过多阴影

### 字体情绪

- **Display性格**：几何无衬线或等宽。推荐：JetBrains Mono / Fira Code /geist Mono
- **Body可读性要求**：
  - 代码：等宽字体，13-15px，行高1.5+，需支持ligature
  - 文档：无衬线，15-16px，行高1.7+
  - 终端输出：等宽，需支持ANSI颜色
  - 关键规则：代码字体和正文字体必须视觉和谐，不能突兀切换

### 关键效果

- **代码高亮与格式化**：语法高亮是底线，不是加分项
- **命令执行反馈**：spinner / 进度条 / 实时日志输出
- **快捷键提示**：kbd标签样式，显示快捷键组合
- **终端/REPL交互**：输入输出视觉区分，支持历史记录
- **错误提示**：精确到行号的错误定位 + 建议修复方案
- **主题切换**：暗/亮主题一键切换，跟随系统偏好

### 反模式

1. **用Landing Page的精美度来掩盖工具本身的功能缺失** — 开发者会试用，不会只看官网
2. **隐藏CLI/API只做GUI** — 开发者需要可编程接口，纯GUI工具会被认为"不专业"
3. **错误信息只有"Something went wrong"** — 开发者需要stack trace、错误码、上下文信息
4. **代码区域使用比例字体** — 这是最快的"不懂开发者"信号
5. **自动播放的营销视频作为主要说明方式** — 开发者更愿意读文档和代码，不是看视频

### 参考案例

- **GitHub** — 开发者平台标杆；signature：代码diff可视化+PR工作流UI
- **Vercel** — 开发者工具标杆；signature：Geist字体+终端式部署日志
- **Raycast** — macOS开发者工具标杆；signature：命令面板交互范式

## 五、教育 / 学习

### 风格倾向

1. **专注沉浸型（推荐）** — 减少视觉干扰，帮助用户进入心流状态。理由：学习需要深度专注，UI应是"隐形助手"而非"注意力竞争者"。
2. **引导渐进型** — 清晰的学习路径可视化，进度感驱动。理由：学习最大的敌人是放弃，可视化的进展能有效维持动力。
3. **活泼启发型** — 适用于K12/兴趣学习，色彩丰富但有序。理由：年轻学习者需要趣味性来维持注意力，但趣味不等于混乱。

### 配色情绪

- **主色调方向**：
  - 成人学习：深蓝 `#1E40AF` / 深绿 `#065F46` — 专注、沉稳
  - K12学习：多色系但统一饱和度 — 活泼但不刺眼
  - 语言学习：品牌色为主，辅以国旗/文化色彩
- **Accent策略**：
  - 正确/完成：绿色 `#22C55E` + 微奖励动效
  - 错误/需复习：暖橙 `#FB923C` — 不要用红色（会引发焦虑）
  - 进度：蓝紫渐变 `#6366F1` → `#8B5CF6`
  - 重点标记：荧光黄 `#FDE047` 背景（类似真实荧光笔效果）
- **避忌**：不要使用红色标记错误——学习场景下红色会引发负面情绪和自我怀疑

### 字体情绪

- **Display性格**：圆润、友好、易读。推荐：Nunito / Source Sans 3 / 思源黑体
- **Body可读性要求**：
  - 学习内容：16-18px（比一般网页更大），行高1.7+
  - 代码练习：等宽14px+
  - 注释/提示：14px，颜色区分
  - 关键规则：阅读舒适度是第一优先级，宁可多翻页也不要小字密排
  - 中文场景：必须考虑字体渲染质量，避免细体在低分屏上发虚

### 关键效果

- **进度可视化**：进度条/步进器/技能树，让学习进展可感知
- **答题反馈**：正确=绿色勾+微庆祝动效，错误=橙色叉+正确答案展示
- **专注模式**：隐藏导航/侧边栏，全屏内容区
- **连续打卡**：日历热力图 + streak计数 + 即将断链提醒
- **手写/公式**：支持LaTeX渲染和手写输入

### 反模式

1. **游戏化过度** — 徽章/排名/连击动效太多会喧宾夺主，学习本身才是核心奖励
2. **进度不可见** — 用户不知道自己学了多少、还差多少，这是放弃的主要原因
3. **错误反馈只说"错了"** — 教育产品的核心价值在于解释"为什么错"和"怎么对"
4. **移动端文字过小** — 很多学习场景在手机上发生（通勤/睡前），字号必须够大
5. **忽视间隔重复的UI** — 记忆类学习需要间隔重复机制，UI应自然引导复习而非只在用户主动时才展示

### 参考案例

- **Duolingo** — 语言学习标杆；signature：绿色猫头鹰IP+streak机制+渐进式课程树
- **Khan Academy** — 知识学习标杆；signature： mastery进度系统+视频+练习三位一体
- **Linear (非教育但交互值得参考)** — 专注模式的交互范式

## 六、金融 / 支付

### 风格倾向

1. **安全信赖型（推荐）** — 视觉传达稳定、专业、可靠，每个元素都在强化"你的钱是安全的"。理由：金融产品用户的首要诉求是安全感，任何轻浮或不确定的视觉元素都会引发焦虑。
2. **清晰透明型** — 信息层级极其清晰，费用/利率/风险无死角展示。理由：监管要求和用户信任都需要透明的信息展示，隐藏=欺骗。
3. **现代简约型** — 传统金融的沉闷+现代设计的简洁，适用于数字银行/支付工具。理由：年轻用户拒绝传统银行的"老气"，但也不能接受过度活泼——需要"专业但不过时"。

### 配色情绪

- **主色调方向**：
  - 银行/储蓄：深蓝 `#1E3A5F` / 深绿 `#065F46` — 安全、增长
  - 支付/转账：品牌色为主（如微信绿、支付宝蓝）
  - 投资/交易：深色 `#0F172A` + 数据可视化配色
  - 加密货币：大胆但专业——可使用渐变但需克制
- **Accent策略**：
  - 收入/盈利：绿色 `#22C55E` — 全球通用，不要替换
  - 支出/亏损：红色 `#EF4444` — 全球通用，不要替换
  - **关键规则**：红绿含义在不同文化中有差异（中国红=涨），需要根据目标市场调整或添加▲▼符号消除歧义
  - 安全/认证：蓝盾色 `#1D4ED8` + 盾牌/锁图标
  - 确认/支付：高对比accent按钮，颜色不用于其他非关键操作
- **避忌**：不要使用过多渐变和玻璃拟态——在金融场景中会显得"虚"

### 字体情绪

- **Display性格**：无衬线、稳重、现代。推荐：Inter / SF Pro / Roboto — 不追求个性，追求信赖
- **Body可读性要求**：
  - 金额数字：24px+，字重700+，等宽数字（tabular figures）
  - 利率/费率：16px+，需标注单位（%/年化）
  - 交易明细：14px，时间戳精确到秒
  - 关键规则：金额必须使用等宽数字，避免数字宽度变化引起的对齐问题

### 关键效果

- **数字滚动动画**：金额变化时数字平滑滚动（countup/countdown），传递"实时性"
- **支付确认流程**：明确的步骤指示 + 不可逆操作的二次确认
- **安全指示器**：SSL锁/认证标志常驻可见
- **交易状态**：处理中=旋转spinner / 成功=绿色勾 / 失败=红色叉+重试按钮
- **指纹/面容识别动画**：支付确认时的生物识别视觉反馈

### 反模式

1. **金额显示过小或视觉权重不足** — 这是用户最关心的信息，必须一眼识别
2. **隐藏手续费或小字说明** — 这是违法的（多数市场），也是最快失去信任的方式
3. **使用过多动画** — 金融场景下频繁动画会让用户觉得"不稳重""像游戏"
4. **盈亏颜色不统一** — 有时红=亏有时红=赚，这是最严重的可用性问题之一
5. **支付按钮不够突出** — "确认支付"按钮必须是页面上视觉权重最高的元素

### 参考案例

- **Revolut** — 数字银行标杆；signature：深紫渐变+卡片动效+清晰资产展示
- **Robinhood** — 投资App标杆；signature：绿色增长线+极简交易流程
- **WeChat Pay** — 支付标杆；signature：绿色品牌色+即时确认+扫码体验

## 七、医疗 / 健康

### 风格倾向

1. **安心专业型（推荐）** — 视觉传达"你被照顾着"，专业但不冰冷，温暖但不轻浮。理由：健康场景下用户往往处于焦虑状态，设计需要在专业感和温度感之间找到平衡。
2. **清晰导向型** — 信息展示无歧义，操作路径零认知负担。理由：医疗信息的误读可能有严重后果，视觉设计必须消除一切歧义。
3. **隐私安全型** — 数据展示克制，隐私保护视觉化。理由：健康数据是最敏感的个人信息，UI需要让用户感知到隐私被尊重。

### 配色情绪

- **主色调方向**：
  - 医院/诊疗：医疗蓝 `#0077B6` / 深青 `#0E7490` — 专业、冷静、信任
  - 健康/健身：活力绿 `#16A34A` / 运动橙 `#EA580C` — 活力、积极
  - 心理健康：薰衣草紫 `#7C3AED` / 暖米色 `#F5F0EB` — 舒缓、安全
  - 中医/养生：大地色 `#8B6914` / 墨绿 `#365314` — 自然、传统
- **Accent策略**：
  - 警告/异常：琥珀色 `#F59E0B` — 不用红色（会引发恐慌）
  - 紧急/危急：红色 `#DC2626` — 仅用于真正紧急的情况
  - 正常/健康：绿色 `#22C55E`
  - 用药/提醒：蓝色 `#3B82F6`
  - 关键规则：颜色分级必须与紧急程度严格对应，不能乱用
- **避忌**：不要使用纯白+红色组合——这会让人联想到血和医院，引发不安

### 字体情绪

- **Display性格**：无衬线、清晰、中性。推荐：Inter / Noto Sans SC / Source Han Sans — 不追求个性，追求零歧义
- **Body可读性要求**：
  - 诊断/处方：16px+，行高1.7+，字间距略宽
  - 数据指标：等宽或比例字体均可，但单位必须明确
  - 药品名称：必须足够大，不能与通用名混淆
  - 关键规则：任何医疗信息都绝不能使用纯装饰性字体或过细字重
  - 中文场景：药品通用名和商品名的字号/字重必须有明确区分

### 关键效果

- **数据趋势图**：生命体征/检查指标的时序可视化，异常值高亮
- **用药提醒**：推送通知 + 剂量确认 + 服药打卡
- **预约流程**：日历选择 + 时段选择 + 医生信息 + 确认 — 步骤清晰
- **隐私遮罩**：敏感信息默认模糊，点击/验证后显示
- **紧急呼叫**：一键触达，按钮足够大且位置固定

### 反模式

1. **使用恐怖/负面视觉元素** — 疾病插图、骷髅图标、血红色警告——这些会加剧焦虑
2. **医学术语不解释** — 非专业用户不理解"收缩压""舒张压"，需要通俗解释或可视化
3. **预约流程过于复杂** — 生病时精力有限，3步内完成预约
4. **忽视色盲用户** — 医疗数据图表必须不依赖颜色区分，使用形状/标签辅助
5. **健康数据没有时间戳** — "血压偏高"没有时间参照毫无意义

### 参考案例

- **Apple Health** — 健康数据标杆；signature：卡片化数据展示+趋势图+隐私优先
- **Zocdoc** — 预约平台标杆；signature：搜索→选择→预约的极简流程
- **Calm** — 心理健康标杆；signature：自然场景+柔和动画+呼吸引导

## 八、内容 / 媒体

### 风格倾向

1. **内容沉浸型（推荐）** — 界面退后，让文字/图片/视频成为绝对主角。理由：媒体产品的核心资产是内容，UI应服务内容阅读和观看，不抢占注意力。
2. **编辑策展型** — 精心编排的内容层次，传达"我们在为你筛选"。理由：信息过载时代，策展能力本身就是产品价值，视觉上需要体现"精选"感。
3. **互动叙事型** — 长内容融入交互元素（数据可视化、时间线、注释）。理由：纯文字长内容的完读率极低，交互元素可以显著提升参与度。

### 配色情绪

- **主色调方向**：
  - 新闻/资讯：品牌色为主，内容区纯白/近白
  - 杂志/长文：品牌色+编辑色板，允许更大胆
  - 视频/播客：深色背景 `#0F0F0F`（视频内容）/ 暖白（文字内容）
  - 知识/百科：极简中性色，信息层级用灰度和排版解决
- **Accent策略**：
  - 链接：标准蓝 `#2563EB` — 这是用户训练过的行为，不要创新
  - 标签/分类：每个分类一个专属色，但饱和度统一
  - 重点引用：左侧竖线 `3px solid 品牌色` + 背景色块
  - 多媒体切换：图标+颜色区分文字/视频/音频
- **避忌**：不要让UI颜色与内容图片冲突——保持中性

### 字体情绪

- **Display性格**：根据内容调性选择——
  - 严肃新闻：传统衬线如Source Serif Pro / Noto Serif SC
  - 科技媒体：几何无衬线如Inter / Space Grotesk
  - 文化杂志：人文衬线如Lora / Crimson Pro
  - 年轻向：大胆display如Bebas Neue / Oswald
- **Body可读性要求**：
  - 长文阅读：16-18px，行高1.7-1.8，最大行宽65-75字符
  - 标题：明确层级 H1(32px+) > H2(24px+) > H3(20px+) > H4(18px+)
  - 引用：斜体或左侧竖线+缩进
  - 图片说明：12-13px，灰色，与正文明确区分
  - 关键规则：中文长文必须使用1.75+行高，段间距大于行间距

### 关键效果

- **阅读进度条**：页面顶部细线进度指示器
- **字体大小调节**：用户可控的字号切换（小/中/大）
- **暗色/亮色/护眼模式**：三种阅读模式切换
- **目录导航**：长文侧边浮动目录，当前章节高亮
- **划线/笔记**：选中文字后的划线+笔记+分享交互
- **沉浸阅读**：隐藏导航，全屏阅读模式

### 反模式

1. **内容区两侧放大量广告/推荐** — 破坏阅读沉浸感，用户会直接关闭
2. **自动播放视频/音频** — 这是用户最讨厌的体验之一
3. **字号不可调** — 不同用户（年龄/视力/设备）需要不同的字号
4. **移动端全文不分页** — 5000字以上的文章在移动端应有分页或"展开剩余"机制
5. **忽视RSS/导出/离线** — 深度读者需要这些功能，忽视他们就是忽视最忠实的用户

### 参考案例

- **Medium** — 长文阅读标杆；signature：极简阅读体验+划线笔记+推荐算法
- **The Verge** — 科技媒体标杆；signature：大胆排版+3D元素+独特配色
- **Pocket** — 稍后阅读标杆；signature：纯净阅读模式+标签分类+离线支持

## 附录：跨行业通用规则

### 排版通用

- 正文字号下限14px（中文13px仅限辅助信息）
- 行高1.5起步，长文1.7+
- 标题字重600-700，正文400，避免500
- 中文段落首行不缩进（现代Web惯例），用段间距区分段落
- 中英文混排时英文前后加空格

### 交互通用

- 可点击元素触控区下限44x44px
- 状态变化必须有视觉反馈（hover/active/focus/disabled）
- 加载状态不能是空白（骨架屏/spinner/进度条）
- 表单验证实时进行，不要等提交后才显示错误
- 不可逆操作需要确认弹窗

### 无障碍通用

- 对比度至少满足 WCAG AA（4.5:1正文，3:1大字）
- 不依赖颜色传达信息（需辅助以文字/图标/形状）
- 支持键盘导航
- 图片需要alt文本
- 动画可关闭（尊重prefers-reduced-motion）

### 配色通用

- 主色+accent色不超过3种饱和色
- 灰度至少6级：50/100/200/400/600/900
- 任何颜色必须通过对比度检验后才能使用
- 暗色模式不是简单反色——需要独立调色
