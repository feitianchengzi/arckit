# 反AI默认风格指南

> 本文档帮助你识别和规避AI生成设计中最常见的三种默认风格，建立项目的独特视觉语言。

---

## 一、三种AI默认风格的详细特征

### 1. 暖奶油风（Warm Cream Default）

这是AI最常输出的风格，几乎没有例外地出现在"现代、优雅、温暖"等宽泛需求下。

**视觉特征：**

| 属性 | 典型值 |
|------|--------|
| 背景色 | `#F4F1EA` / `#FAF7F2` / `#F5F0E8` — 偏黄的暖白色 |
| 主文字色 | `#2D2A26` / `#3A3632` — 不纯黑的暖深棕 |
| Accent色 | 赤陶色 `#C4724E` / `#B86444` / 焦糖色 `#D4956A` |
| 次级色 | 鼠尾草绿 `#8B9D77` / 暖灰绿 `#A3B59A` |
| Display字体 | 衬线体：Playfair Display / Cormorant Garamond / DM Serif Display |
| Body字体 | 无衬线：Inter / DM Sans / Nunito Sans |
| 圆角 | `8px - 12px`，柔和但不极端 |
| 阴影 | `0 2px 8px rgba(0,0,0,0.06)` — 极淡投影 |
| 间距 | 宽松，`24px - 48px` section gap |

**标志性布局：**
- 居中大标题 + 衬线display
- 左右对称的双栏图文排列
- 卡片带极淡边框 `1px solid rgba(0,0,0,0.06)`
- CTA按钮为圆角实心赤陶色
- 页面顶部留大量空白（hero section的"呼吸感"）

**为何AI偏爱：**
- 暖色调在情绪上几乎不会出错
- 衬线+无衬线组合是"优雅"的快捷公式
- 低饱和度配色对非专业需求者显得"高级"

---

### 2. 暗黑酸色风（Dark Acid Default）

当需求中出现"科技感""未来感""赛博朋克"等关键词时，AI几乎100%输出此风格。

**视觉特征：**

| 属性 | 典型值 |
|------|--------|
| 背景色 | `#0A0A0A` / `#111111` / `#0D0D0D` — 接近纯黑 |
| 主文字色 | `#E8E8E8` / `#F0F0F0` — 高对比冷白 |
| Accent色1 | 酸绿 `#BFFF00` / `#C6FF00` / `#AAFF32` |
| Accent色2 | 朱红/品红 `#FF2D55` / `#FF0055` / 霓虹蓝 `#00D4FF` |
| Display字体 | 等宽/几何：Space Grotesk / JetBrains Mono / Syne |
| Body字体 | Inter / Space Mono / IBM Plex Mono |
| 圆角 | `0px - 4px` — 极小或无圆角 |
| 阴影 | 发光效果：`0 0 20px rgba(191,255,0,0.3)` |
| 分割线 | 细亮线：`1px solid rgba(191,255,0,0.2)` |

**标志性布局：**
- 网格线背景或点阵背景
- 终端/代码风格的文字块
- "闪烁"或"扫描线"动效
- Neon glow效果的按钮和边框
- 大面积黑色 + 零星高饱和度色块
- 等宽字体的数据展示区

**为何AI偏爱：**
- "深色=科技"是最简单的语义映射
- 酸绿/朱红组合在深色底上视觉冲击力强
- 等宽字体自带"代码/黑客"联想
- 效果模板化程度高，容易批量产出

---

### 3. 报纸排版风（Newspaper Editorial Default）

当需求涉及"内容平台""媒体""严肃""权威"时出现。

**视觉特征：**

| 属性 | 典型值 |
|------|--------|
| 背景色 | 纯白 `#FFFFFF` 或新闻纸色 `#F8F5F0` |
| 主文字色 | 纯黑 `#000000` 或深墨 `#1A1A1A` |
| Accent色 | 极少，偶见正红 `#E60000` 或深蓝 `#003366` |
| Display字体 | 超粗衬线：Fraunces / Newsreader / Lora Bold |
| Body字体 | 传统衬线：Source Serif Pro / Lora / Merriweather |
| 圆角 | `0px` — 绝对零圆角 |
| 分割线 | 极细：`1px solid #000` / `0.5px solid rgba(0,0,0,0.3)` |
| 栏数 | 3-4栏密集分栏布局 |

**标志性布局：**
- 极细黑色分割线分隔区域
- 多栏文本密集排列
- 上标或drop cap首字母
- 极窄margins（`16px - 24px`）
- 小字号密集信息展示
- 黑白为主，颜色仅用于极少数强调
- "打破网格"的不规则图片裁切

**为何AI偏爱：**
- 衬线+细线+多栏是"编辑/出版"的视觉速记
- 极简配色让AI不需要处理色彩决策
- "严肃感"通过克制来体现，执行路径简单

## 二、自检清单

在完成设计后，逐条回答以下问题。如果有3个及以上回答为"是"，你的设计很可能踩中了AI默认风格：

### 通用自检

1. **如果你的页面背景色是 `#F4F1EA` / `#FAF7F2` 或接近的暖白，并且accent色落在赤陶/焦糖色范围内——你是否只是因为"看起来舒服"而选择它，而非基于项目语义？**

2. **你的display字体是否是 Playfair Display / Cormorant Garamond / DM Serif Display 之一？如果是，你是否考察过至少5个替代字体后才选定？**

3. **你的深色主题是否使用了酸绿 `#BFFF00` 或朱红 `#FF2D55` 作为accent？这两个颜色在你的项目中是否有明确的语义来源（而非"看起来酷"）？**

4. **你的页面是否使用了"居中大标题+两侧留白+单CTA按钮"的hero布局？这是AI的默认模板布局，你是否考虑过其他信息架构？**

5. **如果你遮住logo和品牌名，你的页面和任意一个同行业的AI生成页面能否区分？如果不能，说明视觉语言不够独特。**

### 风格专项自检

**暖奶油风专项：**
- 你的配色方案是否能在Pinterest上找到100个相似案例？
- 你的CTA按钮是否是赤陶色圆角实心按钮？
- 你的卡片是否都带有极淡边框和微弱投影？

**暗黑酸色风专项：**
- 你的深色页面是否使用了网格线/点阵背景？
- 你的accent颜色是否只在"酸绿/朱红/霓虹蓝"中挑选？
- 你的按钮是否带有发光效果？

**报纸排版风专项：**
- 你的页面是否全部使用衬线字体？
- 你的分割线是否都是1px纯黑实线？
- 你的信息密度是否过高，以至于移动端几乎不可用？

## 三、规避策略

### 规避暖奶油风的替代方向

| 替代方向 | 描述 | 参考情绪 |
|----------|------|----------|
| **冷调石材感** | 用冷灰白 `#EDEEF0` 替代暖白，搭配石灰/水泥纹理，accent用冷调金属色如青铜 `#8B7355` 或氧化铜 `#5F8A8B` | 安静、克制、有物质感 |
| **日式和纸感** | 用和纸色 `#F0EDE5` 替代奶油白，但accent不用赤陶而用靛蓝 `#264653` 或�的矢色 `#8B6F4E`，字体选用Noto Serif JP等和风衬线 | 文雅、留白、东方气质 |
| **大地色重彩** | 保留暖调但加深底色至沙色 `#E8DCC8`，accent使用深沉的赭石 `#8B4513` 或苔绿 `#4A5D3A`，而非轻飘的赤陶 | 厚重、自然、有力量感 |
| **北欧冷木感** | 近白底色但偏冷 `#F5F5F0`，accent用深松绿 `#2F4F4F` 或桦木棕 `#8B7D6B`，字体选用几何无衬线如Avenir/FF Mark | 清冷、功能、北欧设计 |

### 规避暗黑酸色风的替代方向

| 替代方向 | 描述 | 参考情绪 |
|----------|------|----------|
| **深蓝工具感** | 背景 `#0F172A`（Tailwind Slate 900）替代纯黑，accent用琥珀 `#F59E0B` 或青色 `#06B6D4`，带功能导向的精确感 | 专业工具、仪表盘、数据分析 |
| **暗色材质感** | 背景 `#1C1C1E` 搭配微妙噪点纹理，accent用暖金 `#C9A96E` 或象牙白 `#FFFFF0`，字体用人文无衬线如FF Meta | 奢侈品暗色、高级定制、质感 |
| **墨绿深海感** | 背景 `#0A1A1F` 深墨绿，accent用珊瑚 `#FF6B6B` 或月白 `#D4E4F7`，搭配流动的有机形态 | 探索、沉浸、有机科技 |
| **灰度工业感** | 背景不用纯黑而用深灰 `#2D2D2D`，accent用信号橙 `#FF6B00` 或工业黄 `#FFD600`，字体用硬朗的几何体如DIN Alternate | 工业、工程、可靠 |

### 规避报纸排版风的替代方向

| 替代方向 | 描述 | 参考情绪 |
|----------|------|----------|
| **杂志编辑感** | 保留衬线display但引入不对称布局、大胆的图片裁切、出血版面，accent色更大胆如洋红 `#E91E63` | 时尚编辑、视觉张力 |
| **学术简洁感** | 用无衬线体如Source Sans Pro，保留多栏但增加行距和留白，accent用学术蓝 `#1565C0` | 学术、严谨、可信赖 |
| **数据可视化感** | 减少长文排版，用图表/数据/图标替代文字密度，交互式信息架构，accent用信息色系 | 信息密度、交互、现代媒体 |

## 四、Signature元素设计方法

Signature元素是项目的"视觉记忆点"——用户在3秒内就能将其与你的品牌关联的独特设计特征。

### 设计步骤

#### Step 1: 语义挖掘

从项目本身出发，找到**非视觉层面的独特性**：
- 这个产品解决了什么独特问题？
- 目标用户有什么独特的行为或身份特征？
- 品牌名/品牌故事有什么可视觉化的隐喻？

**示例：**
- 一个叫"潮汐"的冥想APP → 海浪节奏、潮汐曲线、渐变蓝绿
- 一个叫"锻造"的开发者工具 → 金属质感、锤击动效、火花accent

#### Step 2: 视觉转译

将语义转化为具体的视觉元素：

| 语义方向 | 可转译的视觉维度 |
|----------|------------------|
| 速度/效率 | 斜切角、渐变方向、动画时长、字体x-height |
| 安全/可靠 | 对称布局、圆角大小、阴影深度、颜色饱和度 |
| 创造/自由 | 不规则网格、手写元素、色彩丰富度、留白变化 |
| 连接/协作 | 线条连接、节点图、重叠元素、共享色域 |
| 生长/进化 | 有机曲线、渐变色、动画弹性、层级扩展 |

#### Step 3: 元素固化

选择1-2个signature元素，在整个产品中**一致且高频地使用**：

- **Signature形状**：如Stripe的倾斜条纹、Notion的emoji图标风格
- **Signature动效**：如Linear的页面过渡、Arc浏览器的标签切换
- **Signature配色**：如Spotify的绿、Tiffany的蓝——但不是简单品牌色，而是独特的使用方式
- **Signature字体组合**：如Vercel的Geist字体家族
- **Signature布局模式**：如Bloomberg Terminal的信息密度模式

#### Step 4: 约束检验

Signature元素必须通过以下检验：
1. **去掉它后，设计是否变得面目模糊？** 如果不是，说明不够独特
2. **竞品能否轻易复制？** 如果只是"加个圆角"级别，说明不够深层
3. **它是否能在5个以上页面/组件中复用？** 如果只在hero用一次，说明不够系统
4. **它是否来自项目语义而非个人偏好？** 如果只是"我觉得好看"，说明缺乏根基

## 五、案例对比

### 案例1：冥想类APP首页

| 维度 | AI默认（暖奶油风） | 独特设计 |
|------|-------------------|----------|
| 背景 | `#FAF7F2` 暖白 | `#0B1426` 深夜蓝 + 微妙星空粒子 |
| 字体 | Playfair Display + Inter | Sora（几何圆润）+ Noto Sans SC |
| Accent | 赤陶色 `#C4724E` | 月光银 `#C8D6E5` + 潮汐青 `#55E6C1` |
| Hero | 居中标题"找到内心的平静" | 实时呼吸动画圆环 + "跟随你的节奏" |
| CTA | 赤陶色圆角按钮"开始冥想" | 渐变呼吸效果按钮，文字随呼吸节奏微变 |
| Signature | 无 | 呼吸圆环动效贯穿全站，频率可自定义 |

**对比结论：** 默认设计传达"温暖但无记忆点"，独特设计传达"沉浸、节奏、个人化"。Signature元素（呼吸圆环）与产品核心功能强绑定。

---

### 案例2：开发者工具Landing Page

| 维度 | AI默认（暗黑酸色风） | 独特设计 |
|------|---------------------|----------|
| 背景 | `#0A0A0A` 纯黑 + 网格线 | `#F8F9FA` 浅灰白 + 代码行号纹理水印 |
| 字体 | Space Grotesk + JetBrains Mono |geist Mono + Inter（反常规的浅色主题） |
| Accent | 酸绿 `#BFFF00` | 调试橙 `#FF6D00` + 断点红 `#FF1744` |
| Hero | "Build Faster" + neon glow | 交互式代码编辑器demo，实时可编辑 |
| CTA | 酸绿发光按钮"Get Started" | 橙色实心按钮 + 点击后出现终端动效 |
| Signature | 无 | 可交互的代码demo嵌入各section；终端风格的通知系统 |

**对比结论：** 默认设计是"看起来像开发者工具"，独特设计是"本身就是开发者体验的一部分"。Signature元素（嵌入式可编辑代码demo）直接展示了产品价值。

---

### 案例3：新闻媒体APP

| 维度 | AI默认（报纸排版风） | 独特设计 |
|------|---------------------|----------|
| 背景 | 纯白 `#FFFFFF` | `#FAFAFA` + 极淡蓝灰纹理 |
| 字体 | Newsreader + Merriweather | 自定义variable font（在serif/sans间滑动） |
| Accent | 正红 `#E60000` | 电光蓝 `#2979FF`（链接/交互）+ 深墨 `#1A1A2E`（标题） |
| 布局 | 4栏密集+1px黑线 | 自适应卡片流 + 阅读时间标签 + 滑动切换频道 |
| 分割 | 1px纯黑实线 | 8px留白间隔 + 极淡色彩标签区分板块 |
| Signature | 无 | "阅读时间"标签系统 + 滑动手势导航 + 文章内AI摘要折叠区 |

**对比结论：** 默认设计模仿纸质报纸，独特设计重新定义数字阅读体验。Signature元素（阅读时间+手势导航+AI摘要）针对移动端阅读场景优化，而非简单移植纸媒美学。

## 六、最终原则

1. **AI默认 ≠ 不好看**，但 = 无辨识度。好看但不独特的设计，在品牌层面是失败的。

2. **规避不是反向操作**。不要因为AI默认用暖白就用纯黑——那只是跳进了另一个默认。关键是在项目语义中找到**唯一正确**的视觉方向。

3. **独特性来自约束**。在充分理解项目语义后，视觉选择会自然收敛到少数几个方向——这些方向就是你的signature的来源。

4. **始终问"为什么"**。每一个视觉决策都应该能追溯到项目需求，而非"AI建议这样做"或"这是当前流行"。

5. **测试辨识度**。把你的设计混入5个竞品截图中，让不了解项目的人指出哪个是你的。如果指不出，回去重新设计。
