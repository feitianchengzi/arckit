# 发布检查清单

> ✅ 必须 / ⭐ 推荐 / 💡 可选

---

## 一、发布前检查

### 代码

- [ ] ✅ 所有代码已合并到发布分支，无遗漏commit
- [ ] ✅ 无debug代码（console.log/print/断点/硬编码测试数据）
- [ ] ✅ 无TODO/FIXME/HACK标记在本次发布范围内
- [ ] ✅ 无注释掉的代码块（git记录了历史，不需要注释保留）
- [ ] ⭐ 无大文件提交（>1MB的非二进制文件）
- [ ] 💡 代码格式化/lint通过

### 测试

- [ ] ✅ 全量单元测试通过
- [ ] ✅ 全量集成测试通过
- [ ] ✅ 核心流程E2E测试通过
- [ ] ✅ 回归测试覆盖变更影响的模块
- [ ] ⭐ 测试覆盖率不低于基线（关键路径100%）
- [ ] ⭐ 性能测试达标（P95延迟<基线×1.2）
- [ ] 💡 安全扫描无Critical/High漏洞

### 配置

- [ ] ✅ 生产环境配置已更新（环境变量/特性开关）
- [ ] ✅ 配置变更已在staging验证
- [ ] ✅ 敏感信息不在代码仓库中（密钥/证书/密码）
- [ ] ⭐ 新增配置项有默认值，不会导致启动失败
- [ ] 💡 配置变更可热更新，无需重启

### 数据库

- [ ] ✅ 迁移脚本已测试且向后兼容
- [ ] ✅ 回滚迁移脚本已准备
- [ ] ✅ 大表变更评估了锁表时间（>1s需停机窗口）
- [ ] ⭐ 数据迁移在staging已跑过
- [ ] 💡 迁移脚本支持幂等执行

### 文档

- [ ] ✅ CHANGELOG已更新
- [ ] ✅ API文档已同步（新增/变更接口）
- [ ] ⭐ 运维手册已更新（新增配置项/新增依赖）
- [ ] 💡 用户文档已更新（新功能使用说明）

### 通知

- [ ] ✅ 相关团队已知晓发布计划（时间/影响范围）
- [ ] ⭐ 客服已知晓可能出现的用户问题
- [ ] 💡 用户公告已准备（如涉及界面/流程变更）

---

## 二、发布中检查

### 部署步骤

- [ ] ✅ 按照部署文档逐步执行，不跳步
- [ ] ✅ 每步完成后检查日志无异常
- [ ] ✅ 数据库迁移先于代码部署执行
- [ ] ⭐ 部署过程有另一个人实时监控
- [ ] 💡 部署过程录屏/记录操作日志

### 灰度策略

| 阶段 | 流量 | 持续时间 | 检查点 |
|------|------|---------|--------|
| Canary | 1-5% | 15-30min | 错误率、延迟、核心指标 |
| Rolling 1 | 25% | 15min | 同上 |
| Rolling 2 | 50% | 15min | 同上 |
| Rolling 3 | 75% | 15min | 同上 |
| Full | 100% | - | 持续监控1小时 |

- [ ] ✅ 灰度每个阶段都执行冒烟测试
- [ ] ✅ 灰度阶段异常立即回滚，不继续扩大流量
- [ ] ⭐ 灰度期间有专人盯盘

### 冒烟测试

- [ ] ✅ 首页正常加载
- [ ] ✅ 登录/注册流程正常
- [ ] ✅ 核心业务流程走通（下单/支付/查询等）
- [ ] ✅ API健康检查端点返回200
- [ ] ⭐ 性能指标在正常范围内

---

## 三、发布后检查

### 监控

- [ ] ✅ 错误率未超过基线+0.1%
- [ ] ✅ P95延迟未超过基线×1.2
- [ ] ✅ CPU/内存使用率正常
- [ ] ✅ 无新产生的告警
- [ ] ⭐ 核心业务指标（转化率/支付成功率）正常
- [ ] 💡 1小时后再次检查

### 日志

- [ ] ✅ 无大量ERROR级别日志
- [ ] ✅ 无异常堆栈信息
- [ ] ⭐ 新功能相关日志正常输出
- [ ] 💡 日志量级与流量匹配

### 用户反馈

- [ ] ✅ 客服无异常工单激增
- [ ] ⭐ 用户社区/社交媒体无负面反馈
- [ ] 💡 主动抽样5-10个用户验证核心流程

## 四、紧急发布流程

> 紧急发布可简化流程，但**门禁不可跳过**。

### 简化项

| 正常流程 | 紧急流程 |
|---------|---------|
| 全量E2E测试 | 核心流程冒烟测试 |
| 完整review | 1人review+1人确认 |
| 灰度5阶段 | 灰度2阶段（5%→100%） |
| 监控1小时 | 监控30分钟 |

### 不可跳过

| 项目 | 原因 |
|------|------|
| 单元测试 | 防止低级错误 |
| 数据库迁移验证 | 防止数据丢失 |
| 回滚方案 | 紧急发布风险更高，更需要回滚准备 |
| 变更通知 | 紧急变更更需要团队知晓 |
| 发布后监控 | 紧急发布的问题发现窗口更短 |
