## 5. 接口定义

### 5.1 [接口名称1]

**方法**: `POST`
**路径**: `/api/v1/[resource]`
**描述**: [接口功能描述]
**鉴权**: Bearer Token

**请求头**:
```
Content-Type: application/json
Authorization: Bearer {token}
```

**请求体**:
```json
{
  "param1": "string",
  "param2": 0,
  "param3": true
}
```

**请求参数说明**:
| 参数名 | 类型 | 必填 | 说明 | 约束 |
|--------|------|------|------|------|
| param1 | string | 是 | [说明] | 最大长度 100 |
| param2 | number | 否 | [说明] | 范围 1-100 |
| param3 | boolean | 否 | [说明] | 默认 false |

**响应示例（成功）**:
```json
{
  "code": 0,
  "data": {
    "id": "uuid-xxxx",
    "field1": "value",
    "createdAt": "2024-01-01T00:00:00Z"
  },
  "message": "success"
}
```

**响应示例（失败）**:
```json
{
  "code": 40001,
  "data": null,
  "message": "参数错误：param1 不能为空"
}
```

**错误码**:
| 错误码 | HTTP状态 | 含义 | 处理建议 |
|--------|---------|------|---------|
| 0 | 200 | 成功 | - |
| 40001 | 400 | 参数错误 | 检查请求参数 |
| 40101 | 401 | 未授权 | 重新登录获取 Token |
| 40301 | 403 | 无权限 | 联系管理员授权 |
| 40401 | 404 | 资源不存在 | 检查资源 ID |
| 50001 | 500 | 服务异常 | 重试或联系支持 |

### 7.2 性能验收

| 指标 | 目标值 | 测试方法 |
|-----|--------|---------|
| 页面首屏加载 | <2s（P95） | Lighthouse / WebPageTest |
| 列表接口响应 | <500ms（P99） | 压测工具 |
| 创建接口响应 | <1s（P99） | 压测工具 |
| 并发用户 | [n] 并发无降级 | JMeter |

### 7.3 安全验收

| 检查项 | 验收标准 |
|--------|---------|
| 鉴权 | 未登录用户访问受保护接口返回 401 |
| 权限 | 越权访问其他用户数据返回 403 |
| 输入校验 | SQL 注入/XSS 攻击被拦截 |
| 敏感数据 | 密码等敏感字段不在响应中明文返回 |
| HTTPS | 所有接口强制 HTTPS |
